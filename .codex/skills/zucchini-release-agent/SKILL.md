# Zucchini Release Agent

Use this skill only for the Zucchini repository.

## Workflow

1. Read [references/release-workflow.md](./references/release-workflow.md) if you need the repo-specific release assumptions or versioning rules.
2. Confirm the working tree is appropriate for release work. The release script expects:
   - a clean git status
   - the current branch to be `main`
   - local `HEAD` to match `origin/main`
   - `gh` to be installed and authenticated
3. Choose the release channel:
   - `prerelease`: create the next `-beta.N` release
   - `main`: publish the stable version for the current prerelease line
4. Run the automation script from the repository root:

```bash
bun .codex/skills/zucchini-release-agent/scripts/create-release.mjs --channel prerelease
```

or:

```bash
bun .codex/skills/zucchini-release-agent/scripts/create-release.mjs --channel main
```

5. Use `--version <semver>` when the default inferred version is not the version you want.
6. Use `--dry-run` first if the user wants a preview instead of making changes.

## Expected Behavior

The automation script should handle the full release flow:

- compute or validate the target version
- update `apps/desktop/package.json`
- regenerate `CHANGELOG.md` with `apps/desktop/scripts/generate-changelog.mjs`
- run `bun run format` after changelog generation
- create `chore: release <version>`
- create tag `v<version>`
- push the release commit and tag to `origin`
- wait for `.github/workflows/release.yml`
- publish the GitHub release from the draft created by CI
- mark beta releases as prereleases and stable releases as latest

## Guardrails

- Do not release from a dirty tree unless the user explicitly asks to change the workflow.
- Do not release from any branch other than `main`.
- Do not bypass the script with ad hoc git and `gh` commands unless the script is broken and you explain why.
- If the GitHub Actions run fails after push, stop, surface the failed run, and do not try to improvise around a broken release.
- Keep the release title as `Zucchini <version>`.
